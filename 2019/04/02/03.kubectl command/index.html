03.kubectl command

Posted on 2019-04-02 | In kubectl , command
Words count in article: | Reading time ≈

kubectl command

command

kubernetes排查问题时经常用到的命令

command description
get 获取列出一个或多个资源的信息。
describe 输出指定的一个/多个资源的详细信息。
logs 输出pod中一个容器的日志
create 指定Yaml或Json，创建资源。
edit 编辑服务器上定义的资源。
rolling-update 执行指定ReplicationController的滚动更新。
delete 删除一个资源。
exec 在容器内部执行命令。
replace 用于对已有资源进行更新、替换。
patch 直接对容器进行打补丁的方式，更改字段数据。
apply 使用方式同replace相同,apply不会删除原有resource,创建新的。
scale 对副本进行扩容或缩小。
autosacle 会给一个rc指定一个副本数的范围
attach 登录到容器，直接查看容器
rollout 回滚

get

通用格式

获取所有namespace

在指定的namespace下获取资源

以yaml格式输出资源

常用的资源类型（{$resourceType}）有

  • po（pod）
  • ns（命名空间namespace）
  • instance（实例）
  • svc（service服务）：定义了一个 Pod 的逻辑分组，一种可以访问它们的策略（微服务）。
  • cm（configMap）：存储全局配置变量的，将分布式系统中不同模块的环境变量统一到一个对象中管理。
  • ds（deamonSet）：在每台计算节点上运行一个守护进程（如日志采集等）。
  • deploy（deployment）：用于启动（上线/部署）一个Pod或者ReplicaSet。

describe

描述一个资源

logs

查看日志

日志追踪

查看指定pod中指定容器的日志

create

通过配置文件名创建一个集群资源对象

edit

可以直接编辑一个资源文件

rolling-update

执行指定ReplicationController的滚动更新。该命令会创建一个新的RC，等新pod完全起来后删除一个旧的pod，之后重复操作，直到替换掉所有的pod。

delete

指定资源类型和名字删除一个资源

利用label,批量删除资源

exec

exec主要作用是在容器内部执行命令

指定容器执行命令

replace

replace命令用于对已有资源进行更新、替换

patch

kubernetes提供了一种在容器运行时，直接对容器进行打补丁的方式，更改字段数据

apply

apply命令的使用方式同replace相同，不同的是，apply不会删除原有resource，然后创建新的。

scale

scale用于程序在负载加重或缩小时副本进行扩容或缩小

autoscale

autoscale命令会给一个rc指定一个副本数的范围

attach

登录到容器，直接查看容器

rollout

查看回滚历史

回滚到上一版本

回滚到指定版本
